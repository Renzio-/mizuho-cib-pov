1 mizuhoBankerBriefing LWC SPEC_04 · Flow A Steps 1-2

Client briefing panel that renders on the Account record page. Displays AI-generated intelligence in three sections: portfolio signals, covenant alerts, and cross-sell triggers.

Purpose

Gives the RM a single scannable view of everything they need before a client meeting — without asking the agent. The LWC acts as the "always-on" intelligence layer while the agent handles interactive Q&A.

Data Sources

SectionObjectKey Fields
Portfolio SignalsMZH_PortfolioSignal__cSignal_Type__c, Amount__c, Maturity_Date__c, Description__c
Covenant AlertsMZH_CovenantAlert__cCovenant_Type__c, Current_Ratio__c, Threshold__c, Status__c
Cross-Sell TriggersMZH_CrossSellTrigger__cProduct_Type__c, Opportunity_Value__c, Rationale__c

Behaviour

  • Loads on Account record page via Lightning App Builder
  • Queries data on connectedCallback() filtered by current Account ID
  • Sections collapse if no data — no empty cards
  • Alert cards show severity badge (High/Medium/Low)
  • Cross-sell cards include a "Discuss with Agent" action button that opens copilot with context

SLDS2 Compliance

  • All styling via var(--slds-*) design tokens
  • Uses lightning-card, lightning-badge, lightning-icon base components
  • Layout via slds-grid / slds-col utilities
  • No hardcoded colours, font sizes, or spacing values
Dependency Requires SPEC_01 (data model) and SPEC_02 (sample data) deployed first. Agent integration (copilot button) requires SPEC_03.
2 Banker Assistant Agent SPEC_03 · Flow A Steps 1-5

The core Agentforce agent. Handles three conversational topics: client briefing generation, meeting capture, and cross-sell intelligence. This is the highest-value component in the demo.

Agent Topics

TopicTrigger PhrasesActions
Client Briefing "Prepare my briefing", "What should I know about [client]", "Brief me on [client]" Query MZH_PortfolioSignal__c, MZH_CovenantAlert__c, MZH_CrossSellTrigger__c, FinancialAccount. Return structured briefing.
Meeting Capture "Capture meeting notes", "Log my meeting", "Meeting summary" Create MZH_MeetingNote__c, create Task records, log Activity. Invoke Post-Meeting Capture flow.
Cross-Sell Intelligence "What products for [client]", "Cross-sell opportunities", "FX products for this maturity" Query MZH_CrossSellTrigger__c, FinancialAccount history. Return product recommendations with rationale.

Grounding Strategy

  • Agent grounded on FSC objects + MZH_* custom objects
  • All responses cite specific records (Account name, amounts, dates)
  • No hallucination — if data doesn't exist, agent responds "I don't have information on that"
  • Context awareness: agent knows current user's portfolio via running user assignment

Actions (Invocable)

ActionTypeInputOutput
generateBriefingApexAccountIdStructured briefing object (signals, alerts, triggers)
captureMeetingNotesFlowAccountId, Summary text, Action itemsMeetingNote ID, Task IDs
getProductRecommendationsApexAccountId, Product categoryList of matched products with rationale

Guard Rails

  • Agent cannot create or modify financial data (read-only on FinancialAccount)
  • Agent cannot access accounts outside current user's coverage assignment
  • All agent actions logged to Event Monitoring for MAS audit compliance
Critical Path This is the most complex component (~3 hrs). Requires data model + sample data deployed first. Test agent responses with real data before demo day.
3 Post-Meeting Capture Flow SPEC_05 · Flow A Step 5

Autolaunched flow triggered by the Banker Assistant Agent after meeting capture. Creates structured records from the agent's parsed meeting notes.

Flow Logic

StepActionDetail
1Create MZH_MeetingNote__cMeeting summary, date, attendees (from agent parsed input)
2Create Task recordsOne per action item extracted. Due dates inferred from text ("by Thursday" → next Thursday). Assigned to named users or coverage team.
3Log ActivityEvent/Task linked to Account + Contact records
4Slack notification (optional)Post to deal room channel if cross-sell action identified
5Schedule follow-upCreate future Event based on agent suggestion or default (1 week)

Input Variables

VariableTypeSource
accountIdTextAgent context (current Account)
meetingSummaryTextAgent-parsed from RM conversation
actionItemsText CollectionAgent-extracted action items
attendeeContactIdsText CollectionMatched from agent conversation or defaults to primary contact
crossSellIdentifiedBooleanWhether a product opportunity was discussed

Error Handling

  • If Slack workspace unavailable: skip notification, log warning, continue
  • If contact not found: create note without attendee link, flag for manual update
  • If task assignee not found: assign to running user with note
Slack Integration Slack notification is non-blocking. If no workspace configured, the flow completes successfully without it. Demo can show this with or without Slack.
4 MZH Data Model SPEC_01 · All Steps

Custom objects and fields that extend FSC standard objects to support portfolio intelligence, covenant monitoring, and meeting capture.

Custom Objects

ObjectLabelKey FieldsRelationship
MZH_PortfolioSignal__c Portfolio Signal Signal_Type__c (picklist), Amount__c (currency), Maturity_Date__c (date), Description__c (text), Severity__c (picklist) Lookup → Account
MZH_CovenantAlert__c Covenant Alert Covenant_Type__c (picklist), Current_Ratio__c (number), Threshold__c (number), Status__c (picklist), Trend__c (picklist) Lookup → Account, Lookup → FinancialAccount
MZH_CrossSellTrigger__c Cross-Sell Trigger Product_Type__c (picklist), Opportunity_Value__c (currency), Rationale__c (long text), Source_Signal__c (lookup) Lookup → Account, Lookup → MZH_PortfolioSignal__c
MZH_MeetingNote__c Meeting Note Summary__c (long text), Meeting_Date__c (datetime), Action_Count__c (number), Next_Meeting__c (date) Lookup → Account, Lookup → Contact

FSC Standard Objects Used

ObjectConfiguration
AccountRecordType: CIB_Client. Custom fields: MZH_Coverage_Team__c, MZH_Relationship_Tier__c
FinancialAccountUsed as-is. Product types via picklist: Bilateral Loan, FX Facility, Trade Finance, Treasury.
AccountAccountRelationUsed as-is for entity hierarchy (parent/subsidiary)
ContactCustom field: MZH_Client_Role__c (Treasurer, CFO, Head of Treasury, etc.)

Picklist Values

FieldValues
Signal_Type__cLoan Maturity, Rate Change, Market Event, Regulatory Change
Severity__cHigh, Medium, Low
Covenant_Type__cDebt/EBITDA, Interest Coverage, Net Leverage, Current Ratio
Status__c (Covenant)Healthy, Approaching, Breached
Trend__cImproving, Stable, Deteriorating
Product_Type__cFX Forward, Interest Rate Swap, Cross-Currency Swap, Trade Finance, DCM
[ASSUMPTION] Object Prefix Using MZH_ prefix. Verify no namespace conflict in the target org before deployment.
